구동 흐름 (edumaps 방식)

GAS doGet 또는 빌드 시 시트 → JSON 스냅샷 → Next/Vercel ISR(revalidate) 또는 Cloudflare Pages로 정적 패치. 조회수·좋아요만 런타임 카운트(KV/D1 또는 별도 시트 append). 짧은 링크 kit.dgedu.link/<shortId>는 kits.shortId 매핑으로 라우팅합니다.

💡
시트가 복잡해질 때의 대안

위지윅 본문(HTML)·항목 순서·이미지가 늘면 시트 관리가 버거워집니다. 그때는 ① 시트 유지 + 빌드 JSON 캐싱, ② Cloudflare D1/KV로 이전(edu-link과 동일 스택), ③ 항목 본문만 마크다운/MDX + Git 관리를 권합니다. 초기엔 시트로 시작해 항목 수가 임계치를 넘으면 ②로 이전하는 경로가 무난합니다.
